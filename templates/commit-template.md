# Шаблон коммита

## Формат коммита

```
<type>(<scope>): <subject>

<body>

<footer>
```

---

## Типы коммитов

- **feat**: Новая функциональность
- **fix**: Исправление бага
- **docs**: Изменения в документации
- **style**: Форматирование, отсутствующие точки с запятой и т.д.
- **refactor**: Рефакторинг кода
- **test**: Добавление тестов
- **chore**: Обновление задач сборки, конфигураций и т.д.

---

## Примеры

### Добавление нового документа
```
docs(architecture): add new-service.md [62]

Добавлен документ описания нового сервиса.
Обновлена статистика в DOCS.md.

Closes #123
```

### Исправление бага
```
fix(validation): correct path checking in validate-docs.sh

Исправлена проверка путей для вложенных папок.
Теперь корректно обрабатываются пути с пробелами.
```

### Новая функциональность
```
feat(hooks): add pre-push validation

Добавлен pre-push hook для полной валидации структуры.
Проверяет:
- Все файлы из DOCS.md существуют
- Нумерация последовательна
- Нет секретов в коммите
```

---

## Правила

1. Subject не длиннее 50 символов
2. Body не длиннее 72 символов на строку
3. Используй imperative mood ("add" не "added")
4. Ссылайся на issues/PRs в footer
5. При добавлении .md файла указывай номер [N]
