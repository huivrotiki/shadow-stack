# Хартбиты и Мониторинг (Heartbeats)

## 💓 Что такое Хартбиты?
Согласно `.agent/crons.md`, каждый долгоживущий сервис (shadow-api, shadow-bot, zeroclaw-http, free-models-proxy, ollama) обязан писать одну строку в `data/heartbeats.jsonl` раз в 60 секунд с параметрами `{ts, service, pid, free_mb, status}`.

## 📝 Формат записи (JSONL)
```json
{"ts": "2026-04-24T04:41:00Z", "service": "shadow-api", "pid": 1234, "free_mb": 3200, "status": "UP"}
{"ts": "2026-04-24T04:42:00Z", "service": "free-models-proxy", "pid": 5678, "free_mb": 3100, "status": "UP"}
```

## ⚙️ Активные сервисы (согласно `ecosystem.config.cjs`)
1. **shadow-api** (:3001) — Express API (ZeroClaw)
2. **shadow-router** (:3002) — Cascade Router
3. **telegram-bot** (:4000) — OpenCode Telegram Bridge
4. **health-dashboard** (:5176) — Vite + React
5. **ollama** (:11434) — Локальные модели (7B, 3B)
6. **free-models-proxy** (:20129) — Barsuk (139 моделей)

## 🚨 Правило Мониторинга
- Один cron (monitor) раз в 3 интервала проверяет пропуски хартбитов.
- Пропуск хартбита ≠ автоматический restart: решение принимает человек.
- Алертинг идёт в Telegram при обнаружении проблем.

## 🔧 Настройка (Конфигурация)
Скрипт отправки хартбитов должен быть добавлен в каждый сервис. Пример для Node.js:
```javascript
setInterval(() => {
  const heartbeat = {
    ts: new Date().toISOString(),
    service: 'shadow-api',
    pid: process.pid,
    free_mb: Math.round(require('os').freemem() / 1024 / 1024),
    status: 'UP'
  };
  require('fs').appendFileSync('data/heartbeats.jsonl', JSON.stringify(heartbeat) + '\n');
}, 60000);
```

## 📍 Логирование
- Файл: `data/heartbeats.jsonl`
- Ротация: еженедельно (настраивается в crons)
- Проверка: `tail -f data/heartbeats.jsonl`
