# Refactor Plan V2 — OmniRouter as Unified Cascade + Notebook LLM + Supermemory

## Context

Архитектура изменилась по факту (см. `shadow-stack_local_1/handoff.md` от 2026-04-04):

- **OpenClaw полностью удалён** (config и `~/.openclaw/`). Термин больше не должен использоваться.
- **OmniRoute работает на `:20128`** — 30 моделей (Kiro, Groq, OpenRouter, HF). Это новый центральный каскад.
- **free-models-proxy на `:20129`** — 18 моделей с fallback, backend для OmniRoute.
- **ZeroClaw daemon `:4111`** — лёгкий Ollama-прокси и Telegram канал.
- **Doppler** управляет всеми 8 секретами (не `.env`).
- Dashboard переехал на `:5175`.

Но в коде и документации — старая картина:
- `CLAUDE.md` (оба репо) упоминает `OpenClaw :18789` как основной cloud-маршрут.
- `agent-factory/openclaw.config.json` описывает старый fallback без OmniRoute.
- `agent-factory/.agent/zeroclaw/config.toml` знает только Ollama.
- `agent-factory/.claude/rules/routing.md` содержит `openclaw :18789` в Delegation Matrix + битую строку Fallback Chain (склеенный мусорный повтор `anthropic…anthropic`).
- `notebooks/` нет нигде; Supermemory без namespaces.

**Цель**: сделать OmniRoute (`:20128`) единственным каскадом для cloud-LLM, а **ZeroClaw `:4111`** — пультом управления через Telegram, откуда ставятся задачи агентам (opencode / claude code / pi-coder / autoresearch) с автоматическим fallback на другие модели через OmniRoute когда usage claude-code закончился. Добавить Notebook-слой и Supermemory namespaces, синхронизировать документацию.

**Приоритет выполнения**: сначала ZeroClaw Control Center (R0), потом всё остальное.

### Phase R-1 — Fix opencode local server (БЛОКЕР, делать первым)

Симптом: opencode не открывается, ошибка "Не удалось связаться с Локальный сервер. Автоматическая повторная попытка...". Это блокер для R0 — ZeroClaw control center должен уметь дёрнуть `opencode` CLI.

**R-1.1** Диагностика (read-only):
```bash
# Проверить, что запускает opencode
which opencode
opencode --version 2>&1 | head

# Найти лог local server
ls -la ~/.opencode/ ~/.config/opencode/ 2>/dev/null
find ~/.opencode ~/.config/opencode -name "*.log" 2>/dev/null | head

# Проверить порт, который opencode пытается дёргать
lsof -i -P -n | grep -i opencode
ps aux | grep -i opencode | grep -v grep

# Проверить конфиг
cat ~/.config/opencode/*.json* 2>/dev/null
cat ~/.config/opencode/opencode.jsonc 2>/dev/null
```

**R-1.2** Типичные причины и фикс:
| Причина | Проверка | Фикс |
|---------|----------|------|
| local server не стартует (порт занят) | `lsof -i :3099` (или порт из конфига) | убить процесс держащий порт; перезапустить |
| LLM провайдер в конфиге недоступен | provider в `opencode.jsonc` = anthropic без API key | переключить на OmniRoute (`http://127.0.0.1:20128/v1`) |
| stale lockfile | `~/.opencode/*.lock` | удалить lock, рестарт |
| upstream API ключ истёк | лог показывает 401 | задать через Doppler `doppler run -- opencode` |
| opencode обновился / schema breaking | версия != конфиг | `opencode config reset` + пере-инициализация |

**R-1.3** Перенастройка opencode на OmniRoute (основной фикс):
В `~/.config/opencode/opencode.jsonc` (или `~/.opencode/config.jsonc`) добавить/заменить provider:
```jsonc
{
  "provider": {
    "omniroute": {
      "type": "openai-compatible",
      "baseURL": "http://127.0.0.1:20128/v1",
      "apiKey": "${OMNIROUTER_API_KEY}",
      "models": {
        "claude-sonnet-4.5": {},
        "groq/llama-3.3-70b-versatile": {},
        "openrouter/auto": {}
      }
    }
  },
  "default": { "model": "omniroute/claude-sonnet-4.5" }
}
```
Это убирает зависимость от Anthropic API ключа (у которого кончились кредиты) и даёт fallback через OmniRoute.

**R-1.4** Рестарт:
```bash
pkill -f opencode 2>/dev/null
doppler run --project serpent --config dev -- opencode
```

**R-1.5** Verification:
- opencode открывается без ошибки "Не удалось связаться".
- Test prompt в opencode: `test` → ответ через OmniRoute.

**R0.2** Создать `agent-factory/server/zeroclaw/control-center.cjs`:
- HTTP server на `:4111` (или взять существующий если уже поднят как daemon).
- Telegram long-polling/webhook — получает команды, парсит.
- Роутинг команд → диспатч в agent (spawn child_process для CLI агентов, HTTP для autoresearch).
- Usage tracking: перед вызовом claude-code читает счётчик (локальный файл `.agent/usage/claude-code.json` + timestamp последней 429), если исчерпан → сразу fallback на `omni` + эквивалент-модель (по mapping-таблице).
- Non-blocking: команда возвращает `task_id`, результат приходит вторым сообщением в Telegram.

**R0.3** Telegram команды (минимальный набор Phase R0):
- `/task <описание>` — универсальная задача, дефолтный агент (`claude-code`).
- `/code <описание>` — принудительно через `claude-code` или `opencode`.
- `/opencode <описание>` — принудительно через opencode CLI.
- `/pi <описание>` — через pi-coder (если установлен).
- `/research <запрос>` — autoresearch (DeerFlow или OmniRoute с search-моделью).
- `/ai <prompt>` — прямой вызов OmniRoute (быстрый чат, без CLI агента).
- `/agents` — список доступных агентов + статус (usage, fallback активен?).
- `/usage` — usage claude-code / OmniRoute провайдеров.
- `/cancel <task_id>` — остановить задачу.

Флаг `--research` в `/task` включает опциональный autoresearch-этап перед кодингом.

**R0.4** Usage/Fallback mapping (`agent-factory/.agent/fallback-map.json`):

## Resolved Decisions

1. **OmniRoute port**: `:20128` (из handoff, сервис уже поднят).
2. **Notebook save policy**: оба режима — авто-сохранение в конце сессии (session-end hook) + ручное `/notebook save` для промежуточных чекпоинтов.
3. **Computer Use stack**: гибрид — `screencapture` + `cliclick` для простых скриншотов/кликов (R7.1, R7.2), Playwright (переиспользовать из `server/shadow-router.cjs`) для браузерных сценариев.
4. **Приоритет**: ZeroClaw Control Center (R0) выполняется первым, до любых других фаз.
5. **Control Center = ZeroClaw**: постановка задач через Telegram идёт в ZeroClaw `:4111`, который сам выбирает агента (claude-code / opencode / pi-coder / autoresearch) и делает fallback на OmniRoute когда usage исчерпан.
