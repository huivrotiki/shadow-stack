# Оптимизация скорости LLM Inference

## Глобальная цель

Ускорить генерацию токенов в Ollama (локальная модель qwen2.5-coder:3b) на Mac mini M1 8GB.

## Целевая метрика

**tokens/sec** (больше = лучше)

**Baseline:** ~15 tokens/sec (текущая скорость)  
**Target:** > 25 tokens/sec (улучшение 66%)

## Правила оптимизации

### Разрешено:
- Менять параметры модели (temperature, top_p, top_k)
- Менять batch size и context length
- Оптимизировать промпт (сокращать system prompt)
- Использовать кэширование KV-cache
- Менять num_ctx (размер контекста)

### Запрещено:
- Использовать сторонние библиотеки (только Ollama API)
- Менять модель (должна остаться qwen2.5-coder:3b)
- Использовать GPU (у нас только CPU на M1)
- Нарушать корректность (ответы должны быть осмысленными)

## Ограничения

### RAM Guard
- Доступная RAM: ~380MB (WARNING level)
- Ollama занимает ~280MB при загрузке модели
- Оставшееся: ~100MB для экспериментов
- **Критично:** Всегда `keep_alive: 0` после генерации (выгружать модель)

### Корректность
- Ответы должны быть связными (не gibberish)
- Длина ответа: 50-100 токенов (для тестирования)
- Промпт: "Write a Python function to calculate fibonacci numbers"

## Контекст задачи

Shadow Stack использует Ollama для локальных LLM запросов через:
- Endpoint: `http://localhost:11434/v1/chat/completions`
- Model: `qwen2.5-coder:3b`
- Текущие параметры: default (temperature=0.7, num_ctx=2048)

Узкие места (предположительно):
1. Большой context window (2048) — можно уменьшить
2. Неоптимальные параметры sampling
3. Отсутствие кэширования system prompt

## Стратегия оптимизации

### Phase 1: Low-Hanging Fruit (итерации 1-10)
- Уменьшить num_ctx (2048 → 1024 → 512)
- Отключить sampling (temperature=0, top_p=1)
- Сократить system prompt

### Phase 2: Advanced (итерации 11-30)
- Экспериментировать с num_predict (ограничить длину ответа)
- Тестировать разные значения num_thread
- Оптимизировать промпт для минимального контекста

### Phase 3: Fine-Tuning (итерации 31-50)
- Подбор оптимальных параметров
- Балансировка скорость/качество
- Профилирование узких мест

## Примечания

- Эксперименты запускаются на реальном железе (Mac mini M1)
- Каждый эксперимент = 5 запросов к Ollama (среднее значение)
- Timeout: 5 минут на эксперимент
- Если RAM < 200MB — ABORT (критический уровень)
