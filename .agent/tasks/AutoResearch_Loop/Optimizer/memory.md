- sync → async для I/O операций
- Кэширование повторяющихся вычислений
- Удаление неиспользуемого кода
- O(n²) → O(n log n) алгоритмы

- Замена структур данных (list → set для поиска)

### Fine-Tuning (работает редко)

## Паттерны провала

### Частые ошибки
- Преждевременная оптимизация без профилирования
- Усложнение кода без измеримого эффекта
- Нарушение корректности ради скорости

### Застревание
- 10+ итераций без улучшения → сменить стратегию
- Метрика колеблется ±5% → достигнут локальный минимум
- RAM Guard блокирует эксперименты → упростить код
